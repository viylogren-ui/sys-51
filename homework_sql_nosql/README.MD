# Домашнее задание к занятию "`Базы данных, их типы`" - `Ренёв Виталий`

### Задание 1: СУБД
`Крупная строительная компания, которая также занимается проектированием и девелопментом, решила создать правильную архитектуру для работы с данными. Ниже представлены задачи, которые необходимо решить для каждой предметной области.`
`Какие типы СУБД, на ваш взгляд, лучше всего подойдут для решения этих задач и почему?`

### Решение 1: СУБД
`Задание 1.1.: Бюджетирование проектов с дальнейшим формированием финансовых аналитических отчётов и прогнозирования рисков. СУБД должна гарантировать целостность и чёткую структуру данных.`

`Решение 1.1.: Реляционная СУБД наилучшим образом подойдет в данном случае, т.к. такие системы обеспечивают четкую табличную структуру данных и по схеме ACID гарантируют целостность БД и доступность данных при работе с ней.`

`Задание 1.2.: Под каждый девелоперский проект создаётся отдельный лендинг, и все данные по лидам стекаются в CRM к маркетологам и менеджерам по продажам. Какой тип СУБД лучше использовать для лендингов и для CRM? СУБД должны быть гибкими и быстрыми.` 

`Решение 1.2.: Под лендинг лучше всего подойдут NoSQL графовые СУБД, которые предлагают гибкую настройку с точки зрения продвижения товаров и услуг, при этом NoSQL СУБД позволяют одновременно обрабатывать значительно выше количества запросов по сравнению с реляционными СУБД. Для CRM также подойдет NoSQL СУБД, т.к. позволит гибко настраивать CRM и значительно масштабировать ее без привязки к жесткой структуре (как в реляционных СУБД).`
`Задание 1.3.: Отдел контроля качества решил создать базу по корпоративным нормам и правилам, обучающему материалу и так далее, сформированную согласно структуре компании. СУБД должна иметь простую и понятную структуру.`

`Решение 1.3.: Для решения задачи создания базы по корпоративным нормам и правилам наилучшим образом подойдет NoSQL документно-ориентированная СУБД либо СУБД типа ключ-значение. Такие системы обладают простой и понятной структурой, легко масштабируются и обладают свойствами высокой производительности и доступности.`

`Задание 1.4.: Департамент логистики нуждается в решении задач по быстрому формированию маршрутов доставки материалов по объектам и распределению курьеров по маршрутам с доставкой документов. СУБД должна уметь быстро работать со связями.`

`1.4. Для департамента логистики в целях решения задачи по быстрому формированию маршрутов доставки материалов по объектам наилучшим образом подойдет NoSQL СУБД графового типа, т.к. за счет определения свойств (веса) узлов сети можно будет выстраивать приоритет, что в данном случае является одним из решающих показателей.`


### Задание 2: Транзакции

`2.1. Пользователь пополняет баланс счёта телефона, распишите пошагово, какие действия должны произойти для того, чтобы транзакция завершилась успешно. Ориентируйтесь на шесть действий.`

### Решение 2: Транзакции

1. `Начало транзакции – необходимо зафиксировать этот процесс, с целью гарантии изоляции данной операции.`
2. `Проверка в базе данных телефонных номеров номера телефона, на который будут переводиться денежные средства (если нет, транзакция прекращается)`
3. `Проверка в базе данных банковских счетов наличия счета, с которого будут списываться денежные средства (если нет, транзакция прекращается)`
4. `Проверка наличия на счете необходимого количества денежных средств, необходимых для списания (если нет, транзакция прекращается)`
5. `Осуществление непосредственно списания денежных средств и зачисления на указанный номер телефона (если ошибка при списании, то транзакция прекращается)`
6. `Проверка зачисления денежных средств на номер телефона, успешное завершение транзакции (если ошибка при проверке зачисления денежных средств, то транзакция прекращается).`


### Задание 3: SQL vs NoSQL

`3.1. Напишите пять преимуществ SQL-систем по отношению к NoSQL.`

### Решение 3: SQL vs NoSQL

1. `Четкая структура данных в виде жестко взаимосвязанных таблиц и однозначно построенных отношений между ними;`
2. `Стандартизированный для всех СУБД язык запросов SQL;`
3. `Высокая степень защиты от ошибок при обращении к информации в реляционных СУБД за счет следования фундаментальным принципам транзакций ACID: атомарность, согласованность, изоляция и долговечность;`
4. `Вертикальное масштабирование системы (за счет увеличения мощности сервера);`
5. `Реляционные СУБД являются более оптимальными для сложных запросов;` 


### Задание 4: Кластеры

`Необходимо производить большое количество вычислений при работе с огромным количеством данных, под эту задачу выделено 1000 машин.` 

`На основе какого критерия будете выбирать тип СУБД и какая модель *распределённых вычислений* здесь справится лучше всего и почему?`

### Решение 4: Кластеры

`Приведите ответ в свободной форме........`
`Для выполнения большого количества запросов с большим количеством данных в горизонтально распределенных системах наилучшим образом подойдут нереляционные NoSQL СУБД. По сравнению с реляционными СУБД основными преимуществами NoSQL СУБД являются:`
1. `возможность использования различных моделей построения СУБД в зависимости от выполняемых задач: иерархические, графовые, сетевые, документо-ориентированные, типа ключ-значение и др. В автоматизированной системе в 1000 ПК будут различные задачи по обработке информации и гибкость подходов в решении данных задач будет одним из решающих аргументов в пользу выбора нереляционных СУБД;`
2. `высокая производительность при большом количестве простых запросов также является более сильной стороной NoSQL СУБД;`
3. `высокая гибкость, возможность дальнейшего масштабирования, работа в различных средах разработки широкого круга специалистов - является также одним из аргументов в пользу выбора NoSQL СУБД с точки зрения дальнейшего развития распределенных систем хранения и обработки информации в большой автоматизированной системе.`
